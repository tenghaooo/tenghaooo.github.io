paging(1)

paging是指把process的address space切成多個等分的塊，每塊稱為一個page，而physical memory也相應地切成等分的塊，稱為page frames，每個frame剛好可以容納一個page。

virtual address

從process的視角來看，有一塊連續的address space，假設space總大小為64bytes，以16bytes為一個page，總共就會有4個page。
而假設整個physical memory為128bytes，總共就有8個frame。

把一個process A的4個page分別放到不同的frame，例如page0->frame3，page1->frame7，page2->frame5，page3->frame2。
從程式的視角來看，地址都是連續的，且地址就是0~63，但這0~63是virtual address。
實際上，當CPU要讀取或寫入某個virtual address(VA)時，會將其轉換成對應的physical address(PA)，例如VA0~15會被轉成PA48~63，因為page0->frame3，而VA16~31會被轉成PA112~127，看似連續的VA其實是分散在physical memory中。

這時候如果還有另一個process B，它也有4個page的address space，VA也是0~63，但是它的4個page就會被放到跟A不同的frame中。
因此A跟B都有0~63的VA，但實際上對應的PA並不同，這樣他倆就不能互相讀取修改對方的資料，達到安全隔絕的效果。
不過也有個情況是兩個process的page對應到同個frame，用來做share code或data。

paging可以簡化free-space management，OS可以有一個free-list，記載著目前free的frame，當有一個process需要放page時，從free-list中拿出四個frame即可。

如果frame不夠了，例如上面的例子，其實放了A後剩下的空間是放不下B的，因為總共8個frame而已，扣除A用的4個frame剩下的空間其實不滿4個frame，因為OS本身也需要佔據memory(可能佔走兩個frame)，又或者再來一個process C的話該怎麼辦，之後再寫。

page table

每個process會有一張自己的page table，記載著自己的page被放在哪些frame中，依照table，virtual page會被轉成physical frame(PF)，完成VA->PA的轉換。
以上面的例子來說，會有一個table寫著VP和PF的對應，page0->frame3，page1->frame7，page2->frame5，page3->frame2。

接下來看VA如何透過table轉成PA。
因為每個process的space是64bytes，所以只需要6個bit來表示VA，把VA拆成兩個部分來看，第一是virtual page number(VPN)，第二是offset。
用前兩個bit(msb)表示page number(因為總共4個page，用2個bit即可)，後四個bit表示offset。
假設有個VA是21，轉成6bits就是010101，前兩個bit代表著VA21在哪一個page，後4個bit代表VA21在該page的offset，也就是VA21在page1內(前兩個bit是01，第二個page)，且offset為0101(從page1開始offset 5)。
可以算一下，page1的起始位址是16，offset 5，因此16+5就是21。

接下來拿01去查表，對應的是PFN(physical frame number)是7，因此VA21對應到的PA就是1110101，前三個bit對應PFN(因為共有8個frame，需要3bits表示)，後四個bit對應offset。

page table entry

page table可以看成是一個array，index就是VPN，因此table[0]的內容就是page0對應的PFN，還有其他相關的flag。

valid bit

valid bit表示該page的轉換是否有效，如果process嘗試存取被標記invalid的page，可能會被OS終止執行。又例如一個4G的address space，以4K為一個page，總共就有2^20個page，但不可能每個page都有使用，而且如果每個page明明沒在用卻又要分配一個frame給它，那很浪費physical memory，因此可以把沒使用的page標記成invalid，這樣也無須分配frame給invalid的page，可以大幅節省physical memory。

protection bit

page的RWX。

present bit

用來標示page是在physical memory還是在disk上，也就是判斷page是不是被swap out了。

dirty bit

判斷page是否有被修改過。

reference bit(accessed bit)

用來track page是否有被存取，可以用來決定哪些page比較常使用，在page replacement的時候會需要這個資訊。

一些問題

page table 很大

前面的例子都是比較簡單的情況，如果address space是4G而page大小是4K的話，page table會有2^20個entry，每一個entry佔4bytes的話，一個table就要大概4MB，process數量一多，會佔掉很多memory。
由於page table很大，因此存在memory中，而不會在MMU等硬體內。

paging 很慢

為了要存取到正確的PA，要先知道page table在哪裡，目前先假設page-table base register(PTBR)存著page table的PA，硬體會先根據PTBR，存取一次page table，再依找到的結果去存取正確的PA，這會有兩次存取memory的過程，每一次讀寫memory都要有一個額外查表的動作，拖慢了速度。

後面會再寫到這兩個問題的解法。
